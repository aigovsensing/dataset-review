# 데이터셋 법적 리스크 검토 (Beagle)

오픈 데이터셋을 회사 내부 사전 리스크 검토 기준으로 분석하는 전문 워크플로우다.
**법률 자문이 아니라 내부 참고 자료**이며, 공식적으로 확인 가능한 자료만 근거로 삼는다.

## 언제 사용하나

다음 중 하나라도 해당하면 이 스킬을 로드한다.

- 특정 데이터셋을 회사에서 써도 되는지(상업적 이용·AI 학습 가능 여부) 판단 요청
- 데이터셋의 라이선스 / 데이터 생성·수집 방식 / 개인정보 포함 여부 검토 요청
- 파생 데이터셋의 원본(Upstream) 리스크 상속 검토
- AI 학습 데이터 무단 활용 관련 소송(CourtListener 등) 리스크 분석
- 논문 제출 목적 예외(Major Company 레퍼런스) 적용 가능 여부 확인

## 정본(Authoritative spec) — 반드시 먼저 읽을 것

이 스킬의 **완전한 검토 지침·판단 규칙·출력 형식은 저장소의 다음 파일이 정본이다.**
작업 시작 전에 반드시 이 파일을 Read 로 읽고, 그 지침을 **그대로** 따른다.

    scripts/system_prompt.md

이 파일은 동시에 Gemini 검토 파이프라인(`scripts/review.py`)의 `system_instruction`
으로도 쓰이므로, **검토 로직·출력 형식의 단일 원본(single source of truth)** 이다.
지침이 바뀌면 이 파일만 고치면 되고, 스킬과 파이프라인이 동일하게 반영된다.
아래 요약과 정본이 어긋나면 **정본(`scripts/system_prompt.md`)을 우선**한다.

## 핵심 원칙 (요약 — 상세는 정본 참조)

- 추측·일반 관행 금지. **공식 자료만** 근거로 사용하고, 확인 안 되면 **"확인 불가"** 로 명시.
- 모든 판단에 근거 URL 을 `([출처](URL))` 마크다운 링크로 표기(확인 불가 시에만 생략).
- 출처 우선순위: 공식 홈페이지 > LICENSE > Terms of Use > 논문 > GitHub > Hugging Face.
  블로그·Reddit·SO·커뮤니티 글은 근거로 쓰지 않는다.
- 최신 공식 자료는 **직접 검색으로 확인**한 뒤 원문 문장을 인용한다.

## 검토 축 (요약)

1. **라이선스** — 조건, 상업적 이용·AI 학습 가능 여부. 파생이면 원본 라이선스까지.
2. **데이터 생성·수집 방식** — 직접수집/생성형 AI/기타 SW, 생성 도구·모델의 이용조건,
   **Upstream(원본) 데이터셋 리스크 상속**(파생·원본 중 더 제한적인 쪽 기준으로 판단).
3. **개인정보 포함 여부** — 얼굴·음성·이름 등. 회사 활용까지 포함하는 동의 범위를
   공식 확인 못 하면 잠재 리스크로 판단.
4. **소송 리스크** — 소송 URL 이 있거나 무단 활용 소송 대상으로 확인되면, 원고의
   침해 입증 근거 강도(강/중/약)를 소장 원문 인용·항 번호와 함께 판단. 없으면 "해당 없음".
5. **논문 제출 예외** — 리스크가 있어 판정이 "추가 검토 필요"/"사용 비권고"인 경우에만,
   2023년 이후 Major Company 논문이 동일 데이터셋을 Train/Test 로 활용했는지 확인.
   **판정 자체는 바꾸지 않는다**(논문 제출 목적에 한정된 조건부 경로).

## 출력 형식 (엄수 — 상세는 정본의 [출력 형식] 절)

- 출력은 반드시 `## 종합의견` 으로 시작하고, 사고 과정·서두·메타 설명은 출력하지 않는다.
- `## 1. 요약 결론` 첫 줄은 반드시 고정 형식 `**판정: <사용 가능 / 추가 검토 필요 / 사용 비권고>**`.
  세 문구 중 하나를 **문자 그대로** 쓴다(후처리가 이 줄로 판정을 인식한다).
- **마크다운 표(파이프 `|`) 금지** — 표로 나타낼 내용은 불릿 목록으로 쓴다.
- 전체 한국어, GitHub 댓글 한도 내 간결하게. 세부 섹션·필드 구분자(` / `)는 정본 형식을 정확히 지킨다.

> 위 요약은 오리엔테이션용이다. **실제 검토·작성은 정본 `scripts/system_prompt.md`
> 전체 지침을 읽고 그대로 따른다.**
